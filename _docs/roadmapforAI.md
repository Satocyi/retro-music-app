# roadmapforAI

この文書は、写真を「記憶の空気」に変換するレトロデバイス型アプリの作業順・優先順位を管理するロードマップです。
実装アイデアを何でも詰め込む場所ではなく、AIが次に何をすべきか迷わないための作業地図です。

---

## 0. 現在地

- 旧方針：レトロ音楽体験アプリ
- 新方針：iPhone写真を、ホイール操作でノスタルジックな空気に変換するアプリ
- 音楽連携は中止
- MusicKit / Spotify / 音楽ファイル / 音楽再生は今後の本線から外す
- 現在の黒ホイールUIは継続利用する
- UIの役割は「音楽選択」ではなく「写真の空気をチューニングする操作盤」に変更する
- まずはAI画像生成ではなく、低コストな写真フィルター方式でMVPを作る
- AI画像変換APIは後続検証対象とし、初期MVPには入れない

## Phase 1: 写真MVP体験設計

目的：何を作るアプリかを絞る。

MVPの中心体験：

写真を選ぶ / 撮る
↓
ホイールで空気を選ぶ
↓
写真の色味・質感が変わる
↓
保存する

検討する調整項目：

- 年代
- 季節
- 時間帯
- 天気
- 場所感
- 記憶の質感

- [ ] 写真入力方法を決める（カメラ撮影 / フォトライブラリ）
- [ ] ホイールで調整する項目を決める
- [ ] 初期プリセットを決める
- [ ] 変換後写真の保存方法を決める
- [ ] AI画像生成を使わないMVP範囲を確定する

---

## Phase 2: 低コスト写真フィルター実装

目的：外部AI APIなしで、iPhone内処理だけで価値が出るか確認する。

初期フィルター候補：

- 1998年 夕方
- 2003年 夏
- 雨の日の帰り道
- 古いデジカメ
- くすんだ緑
- 低彩度フィルム
- 夜の室内

- [ ] 写真読み込みUIを作る
- [ ] Core Image 等で色味・粒状感・周辺減光を調整する
- [ ] ホイール操作とフィルター値を接続する
- [ ] 変換前 / 変換後の比較表示を作る
- [ ] 保存機能を作る
- [ ] 実機で処理速度を確認する

---

## Phase 3: レトロデバイスUIの再定義

目的：現在の黒ホイールUIを写真チューニング用に再利用する。

- [ ] LCD表示を音楽/記憶カード前提から写真調整項目へ変更する
- [ ] ホイールで選択中項目を切り替える
- [ ] OKボタンで変換を確定する
- [ ] 変換中の短い演出を作る
- [ ] 写真プレビュー領域とデバイスUIの関係を決める
- [ ] 現在の MemoryTunerView.swift を写真用途に合わせて整理する

---

## Phase 4: AI画像変換の検証 【後回し】

目的：写真フィルターだけでは弱い場合に、AI画像変換を検証する。

- [ ] AI画像変換API候補を調べる
- [ ] 1枚あたりの原価を試算する
- [ ] APIキーを安全に扱うサーバー構成を検討する
- [ ] 無料回数制限を設計する
- [ ] 課金方式を検討する
- [ ] App Store審査上の注意点を確認する

重要：

AI画像変換は初期MVPには入れない。
まずはiPhone内の写真フィルターで成立するかを確認する。
