# COLLABORATION_PROTOCOL

この文書は、このアプリ開発プロジェクトにおけるAI作業ルールです。  
AIはこの文書を「参考」ではなく「作業手順」として扱う。

---

## 1. セッション開始時の必須手順

AIは作業開始時に、以下の順番で読む。

1. `_docs/SYSTEM_CORE.md`
2. `_docs/COLLABORATION_PROTOCOL.md`
3. `_docs/SYSTEM_CURRENT.md`
4. `_docs/roadmapforAI.md`
5. `Handover/ai_handover/` の最新handover

読み終えたら、作業開始前に以下を出す。

```md
## 引き継ぎレポート

### 1. 前セッションから引き継いだこと
### 2. 今セッションで最初にやるべきこと（1つだけ）
### 3. 不明点・確認事項
```

## 2. Core / Current / Handover の役割

| ファイル | 役割 |
| -------- | ---- |
| SYSTEM_CORE.md | 不変の原則 |
| SYSTEM_CURRENT.md | 現在地・実装状態 |
| roadmapforAI.md | 今後の課題・優先順位 |
| Handover | 前回作業の引き継ぎ |

AIは、Coreに反する提案をしてはいけない。
Currentに書かれていない事実は推測しない。
不明なことは「不明」と書く。

## 3. 正本ファイルと作業ファイル

以下は正本ファイルとして扱う。

- `_docs/SYSTEM_CORE.md`
- `_docs/SYSTEM_CURRENT.md`
- `_docs/COLLABORATION_PROTOCOL.md`
- `_docs/roadmapforAI.md`
- `README.md`
- `AGENTS.md`

正本ファイルを変更する場合は、必ずサトシの承認を得る。

## 4. 複数AIの役割

| 役割 | 担当 |
| ---- | ---- |
| 最終判断 | サトシ |
| 設計・方針整理 | Claude |
| アイデア出し・整理 | ChatGPT |
| 実装 | Cursor |
| 監査・環境確認 | 必要に応じてGemini等 |

AIは自分の役割外の判断を勝手にしない。

## 5. 変更前チェック

AIは変更前に以下を確認する。

- Coreに反していないか
- 正本ファイルを触る変更ではないか
- UXの核を変える変更ではないか
- 外部API・課金・保存方針に関わらないか
- サトシの判断が必要ではないか

1つでも不明なら、作業を止めて確認する。

## 6. 判断が必要なもの

以下はAIが勝手に決めない。

- アプリの体験コンセプト
- UXの核
- 課金・広告・ビジネスモデル
- 外部API・外部ライブラリ追加
- 音楽ファイルや著作権に関わる扱い
- ユーザーデータ保存方針
- アプリ名・ブランド名
- 配布方針

該当する場合は、必ず 【判断が必要】 と明示する。

## 7. 作業の進め方

1. 1ステップずつ進める
2. 複数作業をまとめて進めない
3. まず目的を確認する
4. 実装後は動作確認する
5. 同じ失敗が2回出たら、実装ではなく手順・設計を疑う
6. フェーズ外の提案をしない

## 8. Handoverルール

Handoverは、サトシが依頼したときだけ作成する。

保存先：

`Handover/ai_handover/`

ファイル名：

- `chatgpt_handover_YYYYMMDD.md`
- `claude_handover_YYYYMMDD.md`

書く内容：

- このセッションで完了したこと
- 現在地
- 次にやること
- 何をやらなかったか
- 触ってはいけないもの
- 不明点

推測や願望は書かない。

## 9. 古いファイルを勝手に削除しない

AIは、不要に見えるファイルを勝手に削除しない。
削除が必要そうな場合は、必ずサトシに確認する。

## 10. このプロトコルの位置づけ

この文書は憲法ではなく、作業プロトコルである。
プロジェクトの実態に合わせて更新してよい。
ただし、更新にはサトシの承認が必要。
