# スロット切替ホイール仕様 v0

この文書は、MemoryCardの「スロット独立操作」体験を固定するための仕様である。
`_docs/SYSTEM_CORE.md`、`_specs/wheel_spec_v0.md`、`_specs/mvp_flow_v0.md` と矛盾させない。

**バージョン**: v0（初版）
**作成日**: 2026-05-24
**適用**: Phase 3 プロトタイプ実装・以降のUI議論のガードレール

---

## 1. このドキュメントの目的

- MemoryCardを「1枚の固定カード」から「独立したスロットの組み合わせ」に変更する設計方針を固定する。
- ホイール1つで複数スロットを操作する仕組みの「体感の優先順位」を言語化する。
- 実装フェーズで「便利なショートカット」や「全スロット同時スクロール」に流れないよう、ガードレールを置く。

---

## 2. スロット構成

ユーザーが組み合わせる記憶の要素は以下の5つとする。

| スロット名 | 表示ラベル | 値の例 |
|---|---|---|
| YEAR | 年 | 1994 / 1997 / 2001 / 2003 / 2007 / 2012 / 2018 |
| SEASON | 季節 | 春 / 夏 / 秋 / 冬 |
| WEATHER | 天気 | 晴れ / 曇り / 雨 / 霧 / 雪 |
| PLACE | 場所 | 家 / 学校 / 街なか / 海辺 / 海外 |
| SCENE | 情景 | ひとり / 友人と / 通勤中 / 深夜 / 朝 |

- 各スロットの値リストはMVP段階では固定とする。
- 値の追加・編集はMVPスコープ外とする。
- スロットの順序（YEAR→SEASON→WEATHER→PLACE→SCENE）はUI上で固定とする。

---

## 3. 操作方式

### 3-1. 基本方針

ホイールは1つのまま維持する。
ホイールが操作する対象スロットを、MENUボタンで切り替える。

```
MENU押下 → アクティブスロットが1つ進む
YEAR → SEASON → WEATHER → PLACE → SCENE → YEAR（ループ）
```

### 3-2. ホイール操作

- ホイールを回すと、現在のアクティブスロットの値だけが変わる。
- 他のスロットの値は変化しない。
- 値はループする（末尾の次は先頭に戻る）。

### 3-3. 中央ボタン（決定）

- 現在の組み合わせを「確定」するアクション。
- 確定後の画面遷移はmvp_flow_v0の「Detail→Playing」に従う。
- MVPでは複雑な確認ダイアログは持たない。

### 3-4. 生成ボタン

- 現在の5スロットの組み合わせをもとに、記憶の生成・再生を開始する。
- 中央ボタン（決定）と役割が近いが、ラベルで意図を明示する。
- MVPではどちらか一方に統一してよい（Cursor実装時にサトシが判断）。

---

## 4. アクティブスロットの表示

### 4-1. 強調ルール

アクティブスロットは以下の方法で視覚的に強調する。

- 行の背景を暗緑で薄く染める
- 左端に縦ラインのアクセントを表示する
- 値テキストを明るい緑色で発光させる（text-shadow）
- 右端にカーソル（▌）を点滅表示する

### 4-2. 非アクティブスロットの表示

- 値テキストは暗く落とす（アクティブ行との差を明確にする）
- カーソルは非表示
- 発光なし

### 4-3. 初見でわかること

アクティブスロットの強調は「初見のユーザーが操作説明なしで理解できる」レベルを目標とする。
チュートリアルより、視覚的な差分で伝える。

---

## 5. 体感の優先順位

`wheel_spec_v0.md` の思想を継承し、以下を優先する。

- **「今どのスロットを触っているか」が指先でわかること**
- スロット切替は「設定画面を開く」ではなく「チューニングつまみを回す」感覚に近いこと
- MENUを押すたびに「対象が1つずれた」という手触りがあること
- ホイールを回した結果、「この値が変わった」と体感で確認できること

---

## 6. AIが勝手に追加しないこと

以下はサトシの明示なしに実装しない。

- 全スロット同時スクロール（「まとめてランダム」など）
- スロットのタップ直接選択を主導線にすること（補助としてはよい）
- スロットの値をユーザーが自由入力できる機能
- スロット数の増減
- スロット順序の変更UI
- 「おすすめ組み合わせ」などのアルゴリズム的提案の前面化

---

## 7. MVPでやらないこと

- スロット値のカスタマイズ（ユーザー追加・編集）
- 組み合わせの保存・履歴
- スロット間のロック（特定スロットを固定してランダム）
- アニメーションのチューニング以外の演出追加
