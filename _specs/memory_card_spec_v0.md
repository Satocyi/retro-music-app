# MemoryCard 仕様 v0

この文書は、このアプリの中心的なコンテンツ単位である **MemoryCard** の構造・表示・体験上の役割を定義する。  
`_docs/SYSTEM_CORE.md`、`_specs/mvp_flow_v0.md`、`_specs/wheel_spec_v0.md` と矛盾させない。

**バージョン**: v0（初版）  
**適用**: MVP フェーズのホイールUI・Moments画面

---

## 1. MemoryCard とは何か

MemoryCard は、このアプリにおける**選択の単位**である。

ユーザーは曲を直接選ぶのではなく、**ノスタルジックな記憶の断片**をホイールで選ぶ。  
その先に音楽体験がつながる。

これは `mvp_flow_v0.md §9` の以下の方針と一致する。

> アルゴリズム的正解より**並びと手触りの偶然性**が製品コンセプトの中心にある。  
> 「棚の記憶」を残すことをMVPの価値の一部とするなら、その棚はMomentsが担う。

MemoryCard は**棚に並ぶ記憶の切れ端**として機能する。

## 2. データ構造

### 2-1. フィールド定義

MemoryCard は以下の3フィールドで構成する。**MVP では3行固定**とする。

| フィールド | 型 | 役割 | 例 |
|---|---|---|---|
| `line1` | String | 年代・季節・時間帯 | `2007年 夏` |
| `line2` | String | 天気・場所・空気感 | `雨` |
| `line3` | String | 行動・記憶・場面 | `高校帰り` |
| `id` | UUID | 識別子 | （自動生成） |

### 2-2. Swift 定義（参考）

```swift
struct MemoryCard: Identifiable {
    let id: UUID
    let line1: String  // 年代・季節・時間帯
    let line2: String  // 天気・場所・空気感
    let line3: String  // 行動・記憶・場面
}
```

### 2-3. MVPでやらないこと（データ観点）

- 音楽ファイルパスや再生情報との直結（MVP では未接続）
- ユーザーによる編集・追加
- サーバーからの取得・同期
- `line4` 以降の追加（3行固定を崩さない）

## 4. 表示仕様（MVP）

### 4-1. 表示方式

MVP では**選択中の MemoryCard 1枚を大きく表示する**。  
前後カードの薄い表示・スタック表現は後のUI調整フェーズで検討する。

### 4-2. 3行の表示ルール

| 行 | 表示上の扱い |
|---|---|
| line1 | 最も大きく・明るく表示する（時間軸の錨） |
| line2 | line1より小さく・やや薄く |
| line3 | line2と同等またはやや薄く |

**禁止**：
- 3行を同じ大きさ・同じ濃さで並べない（単なる箇条書きに見えてしまう）
- ラベル・アイコン・タグ等の装飾を付けない
- フォントを複数種類混在させない（ウェイト差のみで表現する）

### 4-3. カードの体感的な役割

MemoryCard は「情報カード」ではなく「**記憶の断片**」として見せる。

- 読ませるより、**一瞬で雰囲気が伝わる**ことを優先する
- ユーザーが「あ、こういう感じ」と思えれば十分
- 説明的・網羅的な文字列は避ける

---

## 5. ホイールとの関係

MemoryCard は `wheel_spec_v0.md` のホイール操作で選ばれる。

- ホイールを1ノッチ回す → 表示する MemoryCard が1枚変わる
- 先頭↔末尾はループする（`wheel_spec_v0.md §4` 改訂済み）
- ホイールの「機械を触っている感覚」と MemoryCard の「記憶の断片感」は**体験として一体**である

**MemoryCard の表示切り替えは、ホイールの重さ・鈍さと合っていること**。  
スナップ的にパッと切り替わるより、**一拍おいて現れる**ような印象が望ましい。  
（具体的なアニメーション実装はUIフェーズで決める）

---

## 6. MVPでやらないこと（表示・体験観点）

- 前後カードの薄い表示・スタック・奥行き表現
- カードをタップして詳細画面に遷移する（Detail画面接続は別タスク）
- カード単位での「お気に入り」「削除」「並び替え」
- ユーザーが記憶を入力・編集する画面
- アニメーションの本格チューニング
- 音楽との紐付け（モック再生との接続は別タスク）

---

## 7. WheelItemからの移行方針

既存の `WheelItem`（`title: String` の1フィールド）を `MemoryCard` に置き換える。

| 現在 | 移行後 |
|---|---|
| `WheelItem` | `MemoryCard` |
| `title: String` | `line1`, `line2`, `line3: String` |
| 英語モックデータ7件 | 日本語モックデータ7件（§3） |

`WheelViewModel` と `WheelView` は、`WheelItem` → `MemoryCard` の置き換えに合わせて修正する。  
ループ実装（端の先頭↔末尾接続）もこのタイミングで同時に行う。

---

## 8. AIが勝手に追加しないこと

- `line4` 以降のフィールド追加
- 音楽ファイル・再生情報との直結
- 英語テキストへの差し戻し
- カード枚数の大幅増加（MVP確認用は7枚で十分）
- 前後カードのスタック・奥行き表現（後フェーズ）
- 外部ライブラリの追加

該当する場合は **`【判断が必要】`** としてサトシに確認する。
